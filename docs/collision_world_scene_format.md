# CollisionWorld Scene CSV Format

AdaSDF-CL v1.14.0-alpha adds a small CSV scene format for multi-object SDF
collision orchestration. The format is intentionally simple and text-based so
it can be generated by scripts, tests, and robotics pipelines without pulling
in a scene graph dependency.

## Header

```text
object_id,name,model_path,samples_path,tx,ty,tz,qw,qx,qy,qz,group,mask,type,enabled
```

## Fields

| Field | Required | Meaning |
| --- | --- | --- |
| `object_id` | yes | Stable integer object id. |
| `name` | yes | Human-readable object name used in reports. |
| `model_path` | yes | Path to a queryable `.sdfbin` model. Relative paths are resolved against the scene file. |
| `samples_path` | broadphase no, sparse/contact yes | Path to a collision sample CSV. Relative paths are resolved against the scene file. |
| `tx,ty,tz` | optional | World translation. Defaults to `0,0,0`. |
| `qw,qx,qy,qz` | optional | World rotation quaternion. Defaults to identity `1,0,0,0`. |
| `group` | optional | Collision group bit mask. Defaults to `1`. Hex values such as `0x1` are accepted. |
| `mask` | optional | Collision acceptance mask. Defaults to `0xffffffff`. |
| `type` | optional | `dynamic`, `static`, or `kinematic`. Defaults to `dynamic`. |
| `enabled` | optional | `true` or `false`. Defaults to `true`. |

## Example

```text
object_id,name,model_path,samples_path,tx,ty,tz,qw,qx,qy,qz,group,mask,type,enabled
0,box_a,cube_compressed.sdfbin,cube_samples.csv,0,0,0,1,0,0,0,0x1,0xffffffff,dynamic,true
1,box_b,cube_compressed.sdfbin,cube_samples.csv,0.2,0,0,1,0,0,0,0x2,0xffffffff,dynamic,true
2,floor,cube_compressed.sdfbin,cube_samples.csv,0,-1,0,1,0,0,0,0x4,0xffffffff,static,true
```

## Tools

```bash
adasdf_world_broadphase scene.csv --out pairs.csv --report broadphase.md
adasdf_world_sparse_collide scene.csv --threshold 0 --early-exit --out hits.csv
adasdf_world_solver_contacts scene.csv --threshold 1e-3 --max-contacts 8 --out contacts.csv
adasdf_benchmark_collision_world scene.csv --mode sparse --repeat 10 --csv world_benchmark.csv
```

`adasdf_world_sparse_collide` returns code `10` when collision is detected.
That is a successful collision status, not a process failure.

## Scope Boundary

CollisionWorld in v1.14.0-alpha is sample-based SDF collision over multiple SDF
objects. Broadphase is deterministic AABB pair filtering. Narrowphase uses the
existing sparse point-to-SDF query path and solver contacts are solver-ready
candidates only.

This release does not implement FCL fallback, an external physics solver,
friction, impulses, Jacobian rows, CCD, ROS/MoveIt integration, pybind11
bindings, or a general scene graph.
